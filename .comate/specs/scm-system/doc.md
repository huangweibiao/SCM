# SCM供应链管理系统开发规格文档

## 一、需求概述

### 1.1 项目目标
基于详细设计方案,开发一个完整的SCM供应链管理系统,覆盖供应商管理、采购管理、库存管理、销售管理、生产/委外管理、物流管理、报表分析等核心模块,实现从订单到交付的全链路管理。

### 1.2 技术栈
- **后端框架**: Spring Boot 3.5.11 + Maven
- **数据库**: MySQL 8.0
- **ORM框架**: MyBatis-Plus
- **缓存**: Redis (可选)
- **认证授权**: JWT + OAuth2 + RBAC
- **前端**: Vue3 + Element Plus (独立项目)
- **部署**: Docker容器化

### 1.3 开发范围
本次开发包含完整后端系统实现,包括:
1. 系统基础架构搭建
2. 数据库设计与初始化
3. 核心业务模块实现
4. 接口开发
5. 安全与权限控制

## 二、系统架构设计

### 2.1 分层架构
- **Controller层**: 接收HTTP请求,参数验证,调用Service层
- **Service层**: 业务逻辑处理,事务控制,跨模块调用
- **Mapper层**: 数据库操作,使用MyBatis-Plus
- **Entity层**: 数据库实体映射

### 2.2 模块划分
```
com.scm
├── common           # 通用模块
│   ├── constant     # 常量定义
│   ├── enums        # 枚举定义
│   ├── exception    # 异常处理
│   └── util         # 工具类
├── config           # 配置类
├── entity           # 实体类
├── dto              # 数据传输对象
├── vo               # 视图对象
├── mapper           # MyBatis Mapper
├── service          # 业务接口
│   └── impl         # 业务实现
├── controller       # 控制器
└── ScmApplication   # 启动类
```

## 四、核心业务模块设计

### 4.1 供应商管理模块

#### 4.1.1 核心功能
- 供应商CRUD操作
- 供应商状态管理(启用/停用)
- 供应商评估/评级
- 供应商列表查询(分页)

#### 4.1.2 接口设计
```
POST   /api/v1/supplier           # 创建供应商
PUT    /api/v1/supplier/{id}      # 更新供应商
DELETE /api/v1/supplier/{id}      # 删除/停用供应商
GET    /api/v1/supplier/{id}      # 查询供应商详情
GET    /api/v1/supplier           # 查询供应商列表(分页)
```

#### 4.1.3 关键业务逻辑
- 创建供应商时生成唯一供应商编码
- 删除供应商时校验是否存在关联的采购订单
- 更新供应商评级

### 4.2 采购管理模块

#### 4.2.1 核心功能
- 采购订单创建/审核/取消
- 采购收货(部分/全部)
- 采购订单查询(分页)
- 采购订单状态流转

#### 4.2.2 接口设计
```
POST   /api/v1/purchase/order           # 创建采购订单
PUT    /api/v1/purchase/order/{id}     # 更新采购订单
DELETE /api/v1/purchase/order/{id}     # 删除采购订单
GET    /api/v1/purchase/order/{id}     # 查询采购订单详情
GET    /api/v1/purchase/order           # 查询采购订单列表(分页)
PUT    /api/v1/purchase/order/{id}/audit    # 审核采购订单
POST   /api/v1/purchase/receipt         # 采购收货
```

#### 4.2.3 关键业务逻辑
- **创建订单**:
  - 验证供应商状态是否启用
  - 验证物料状态是否启用
  - 计算订单金额(含税)
  - 生成唯一采购订单号

- **审核订单**:
  - 验证订单状态(仅待审核状态可审核)
  - 状态流转: 10待审核 → 20已审核
  - 记录审核人和审核时间

- **收货流程**:
  - 创建入库单(类型10采购入库)
  - 支持部分收货,更新`received_qty`和`remain_qty`
  - 入库单确认后触发库存增加

- **订单完成**:
  - 所有明细全部收货后,状态自动改为40已完成

### 4.3 库存管理模块

#### 4.3.1 核心功能
- 库存入库/出库
- 库存查询(分页)
- 库存预警查询
- 库存流水查询
- 库存盘点(简化版)

#### 4.3.2 接口设计
```
GET    /api/v1/inventory               # 查询库存列表(分页)
GET    /api/v1/inventory/{id}         # 查询库存详情
GET    /api/v1/inventory/warn         # 查询库存预警
GET    /api/v1/inventory/log          # 查询库存流水(分页)
```

#### 4.3.3 关键业务逻辑
- **库存入库**:
  - 使用`@Transactional`声明事务
  - 通过`select for update`加行锁
  - 使用乐观锁更新库存(version字段)
  - 记录库存流水

- **库存出库**:
  - 校验可用库存(available_qty)
  - 使用行锁+乐观锁保障一致性
  - 库存扣减成功后记录流水

- **库存锁定**:
  - 销售订单审核时锁定库存(增加`locked_qty`)
  - 出库时释放锁定(减少`locked_qty`)

- **库存预警**:
  - 定时任务扫描库存表
  - 判断可用库存 < min_stock 或 > max_stock
  - 触发预警通知

### 4.4 销售管理模块

#### 4.4.1 核心功能
- 销售订单创建/审核
- 销售发货(部分/全部)
- 销售订单查询(分页)
- 销售订单状态流转

#### 4.4.2 接口设计
```
POST   /api/v1/sales/order            # 创建销售订单
PUT    /api/v1/sales/order/{id}       # 更新销售订单
DELETE /api/v1/sales/order/{id}       # 删除销售订单
GET    /api/v1/sales/order/{id}       # 查询销售订单详情
GET    /api/v1/sales/order            # 查询销售订单列表(分页)
PUT    /api/v1/sales/order/{id}/audit     # 审核销售订单
POST   /api/v1/sales/shipment         # 销售发货
```

#### 4.4.3 关键业务逻辑
- **创建订单**:
  - 验证物料状态
  - 验证仓库状态
  - 计算订单金额
  - 生成唯一销售订单号

- **审核订单**:
  - 验证订单状态
  - **校验可用库存**(库存不足则审核失败)
  - 库存充足则锁定库存(`locked_qty`增加)
  - 状态流转: 10待审核 → 20已审核

- **发货流程**:
  - 创建出库单(类型10销售出库)
  - 支持部分发货,更新`shipped_qty`和`remain_qty`
  - 出库单确认后:
    - 扣减库存(`qty`减少)
    - 释放锁定(`locked_qty`减少)
    - 记录库存流水

- **订单完成**:
  - 所有明细全部发货后,状态改为40已完成

### 4.5 生产/委外管理模块

#### 4.5.1 核心功能
- 生产工单创建/计划/执行/完工
- 生产领料
- 产成品入库
- 工单查询(分页)

#### 4.5.2 接口设计
```
POST   /api/v1/production/order       # 创建生产工单
PUT    /api/v1/production/order/{id}  # 更新生产工单
GET    /api/v1/production/order/{id}  # 查询工单详情
GET    /api/v1/production/order      # 查询工单列表(分页)
PUT    /api/v1/production/order/{id}/start    # 开始生产
PUT    /api/v1/production/order/{id}/finish  # 完工确认
```

#### 4.5.3 关键业务逻辑
- **创建工单**:
  - 指定产成品物料
  - 计划生产数量
  - 计划开始/结束日期
  - 生成唯一工单号

- **生产领料**:
  - 创建出库单(类型20生产领料)
  - 出库确认后扣减原材料库存

- **开始生产**:
  - 状态: 10计划 → 20生产中
  - 记录实际开始时间

- **完工入库**:
  - 创建入库单(类型20生产入库)
  - 入库确认后增加产成品库存
  - 更新工单`finished_qty`
  - 所有计划数量完工后状态改为30完工

## 五、接口规范

### 5.1 统一响应格式
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {},
  "timestamp": 1775374295000
}
```

### 5.2 状态码定义
- 200: 成功
- 400: 参数错误
- 401: 未认证
- 403: 无权限
- 404: 资源不存在
- 500: 系统错误

### 5.3 分页参数
```
pageNum: 页码(从1开始)
pageSize: 每页条数
```

### 5.4 分页响应
```json
{
  "total": 100,
  "list": []
}
```

## 六、安全设计

### 6.1 认证授权
- 使用JWT Token认证
- 基于RBAC的权限控制
- 用户、角色、权限三级管理

### 6.2 数据安全
- 密码加密存储(BCrypt)
- 敏感数据脱敏
- 操作审计日志

### 6.3 接口安全
- 所有业务接口需携带JWT Token
- 接口限流(可选)
- XSS防护

## 七、异常处理

### 7.1 业务异常
- 业务校验失败(库存不足、参数错误等)
- 自定义业务异常类`BusinessException`

### 7.2 系统异常
- 数据库异常
- 网络异常
- 统一异常处理器`GlobalExceptionHandler`

## 八、定时任务

### 8.1 库存预警任务
- 执行频率: 每小时
- 扫描库存表,检查低库存/超库存
- 生成预警记录(简化版:日志记录)

## 九、代码规范

### 9.1 命名规范
- 类名: 大驼峰(PascalCase)
- 方法名/变量名: 小驼峰(camelCase)
- 常量: 全大写下划线分隔(UPPER_SNAKE_CASE)
- 数据库表名: 小写下划线分隔(snake_case)

### 9.2 注释规范
- 类注释: 说明类的职责
- 方法注释: 说明方法功能、参数、返回值
- 复杂业务逻辑添加行内注释

### 9.3 包结构规范
- 按业务模块分包
- 遵循单一职责原则

## 十一、边界条件与异常处理

### 11.1 库存相关
- 库存不足时拒绝出库
- 乐观锁更新失败时重试(最多3次)
- 负数库存不允许

### 11.2 订单相关
- 已审核订单不允许删除
- 订单状态流转校验
- 订单金额计算精度问题

### 11.3 数据完整性
- 删除前校验关联数据
- 外键约束处理
- 数据脱敏展示

## 十二、性能优化

### 12.1 数据库优化
- 合理添加索引
- 分页查询避免全表扫描
- 批量操作提升性能

### 12.2 缓存优化
- 物料信息缓存
- 数据字典缓存
- 库存快照缓存(可选)

### 12.3 代码优化
- 避免N+1查询
- 使用批量插入/更新
- 异步处理非核心逻辑(可选)
