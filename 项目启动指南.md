## 一、后端启动步骤

### 1. 数据库初始化

```bash
# 进入后端项目目录
cd scm-backend

# 创建数据库和表结构
mysql -u root -p < src/main/resources/db/schema.sql

# 初始化数据
mysql -u root -p scm_db < src/main/resources/db/init-data.sql
```

### 2. 修改配置文件

编辑 `src/main/resources/application.yml`，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/scm_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: your_password  # 修改为你的MySQL密码
```

### 3. 安装依赖并启动

```bash
# 安装依赖
mvn clean install

# 启动项目
mvn spring-boot:run
```

### 4. 验证后端启动

访问以下地址验证：

- **健康检查**: http://localhost:8080/api/health
- **接口文档**: http://localhost:8080/api/doc.html

默认管理员账号：
- 用户名：`admin`
- 密码：`admin123`

## 六、常见问题

### 1. 后端启动失败

**问题**: 数据库连接失败
```
Communications link failure
```

**解决方案**:
- 检查MySQL服务是否启动
- 检查 `application.yml` 中的数据库配置是否正确
- 确认MySQL用户名和密码是否正确

**问题**: 端口被占用
```
Port 8080 was already in use
```

**解决方案**:
- 修改 `application.yml` 中的端口号
- 或关闭占用8080端口的程序

### 2. 前端启动失败

**问题**: 依赖安装失败
```
npm ERR! code ENOENT
```

**解决方案**:
```bash
# 清除缓存
rm -rf node_modules package-lock.json
# 重新安装
npm install
```

**问题**: 端口被占用
```
Port 3000 is already in use
```

**解决方案**:
```bash
# 指定其他端口
npm run dev -- --port 3001
```

### 3. 前后端联调问题

**问题**: 前端请求后端接口失败

**检查**:
1. 后端是否正常启动（访问 http://localhost:8080/api/health）
2. 前端Vite代理配置是否正确（vite.config.js）
3. 浏览器控制台是否有跨域错误

**解决方案**:
- 确保后端和前端都正常启动
- 检查网络请求地址是否正确
- 查看浏览器控制台和后端日志

### 4. 登录失败

**问题**: 登录时提示"用户名或密码错误"

**检查**:
- 数据库初始化脚本是否执行成功
- 数据库中是否存在默认用户

**解决方案**:
```sql
-- 查询用户
SELECT username, real_name FROM sys_user WHERE username = 'admin';

-- 如果不存在，重新执行初始化脚本
mysql -u root -p scm_db < src/main/resources/db/init-data.sql
```

## 七、开发建议

### 后端开发
1. 使用IDEA的代码生成功能快速创建Mapper、Service、Controller
2. 遵循RESTful API设计规范
3. 所有业务操作使用事务注解 `@Transactional`
4. 涉及库存的操作必须使用乐观锁

### 前端开发
1. 使用Element Plus组件库快速开发
2. 统一使用Composition API（`<script setup>`）
3. 使用Pinia进行状态管理
4. 所有异步操作添加loading状态

### 调试技巧
1. **后端**: 使用IDEA的Debug模式进行断点调试
2. **前端**: 使用浏览器DevTools查看网络请求和Console日志
3. **数据库**: 使用MySQL Workbench或Navicat查看数据
